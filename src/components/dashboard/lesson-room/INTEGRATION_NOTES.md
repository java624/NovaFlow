# Що додано і як підключити

## Нові файли
Постав ці файли в `./lesson-room/` поруч з рештою під-компонентів:

- `ReactionsOverlay.tsx` — емодзі-реакції (кнопка + анімація)
- `PollWidget.tsx` — опитування/квізи
- `LessonTimer.tsx` — таймер уроку
- `ParticipantsPanel.tsx` — список учасників + модерація вчителя
- `useVirtualBackground.ts` — хук для розмиття/заміни фону камери
- `VirtualBackgroundControls.tsx` — кнопка/попап вибору фону

Головний `LessonRoom.tsx` — оновлений, інтегрує все вище.

## 1. Встановити пакет для віртуального фону

```bash
npm install agora-extension-virtual-background
```

Розширенню потрібні `.wasm`-файли в рантаймі. Скопіюй їх з
`node_modules/agora-extension-virtual-background/wasms/` у
`public/agora-vb-wasm/` (шлях уже прописаний в `useVirtualBackground.ts`,
константа `assetsPath`). Якщо хочеш зберігати фонові зображення — постав
JPG-файли у `public/backgrounds/` (`classroom.jpg`, `library.jpg`,
`abstract.jpg`) або зміни список `SAMPLE_BACKGROUNDS` у
`VirtualBackgroundControls.tsx` на свої.

## 2. Роль користувача (вчитель/учень)

`LessonRoom` тепер приймає необов'язковий проп:

```tsx
<LessonRoom
  channelName={channelName}
  userName={userName}
  userRole="teacher" // або "student" (за замовчуванням — "student")
  onLeave={handleLeave}
/>
```

Це не змінює твій існуючий `types.ts` — розширення типу зроблено локально
всередині `LessonRoom.tsx` (`ExtendedLessonRoomProps`). Якщо хочеш, онови й сам
`LessonRoomProps` у `types.ts`, додавши:

```ts
userRole?: 'teacher' | 'student';
```

## 3. Кнопки у `LessonRoomFloatingControls` — тепер справжні

Після того, як ти показав реальний вміст усіх під-компонентів, я оновив і їх:

- **`LessonRoomFloatingControls.tsx`** — додано дві робочі кнопки: 📊 Опитування
  (з зеленою точкою, коли опитування активне, а панель закрита) і 👥 Учасники
  (з лічильником людей у кутку значка).
- **`LessonRoomHeader.tsx`** — додано бейдж «👑 Вчитель» біля назви уроку, коли
  `isTeacher` true.
- **`LessonRoomVideoArea.tsx`** — це найважливіше виправлення: раніше
  компонент рендерив **тільки `remoteUsers[0]`**, тобто відео-зона фізично
  підтримувала лише 1-на-1 урок. Якщо в кімнаті троє й більше (вчитель +
  кілька учнів — а саме для цього зроблено kick/mute all), третій і подальші
  учасники просто не мали свого відео-тайла. Тепер:
  - у режимі **"Фокус"**: показує спікера, що зараз говорить (`activeSpeakerUid`),
    а решту учасників — маленькою стрічкою мініатюр згори;
  - у режимі **"Сітка"**: рендерить по одній картці на кожного учасника
    (адаптивна кількість колонок — 1, 2, 2×2, 2×3, 3×4 залежно від кількості людей);
  - для кожного remote-тайла тепер показується справжнє ім'я та бейдж
    "Вчитель" (раніше — просто напис "Співрозмовник"), а також іконка 🔇, якщо
    в учасника вимкнено мікрофон. Це стало можливим завдяки новому пропу
    `participantProfiles` (uid → ім'я/роль), який головний файл уже передає.
- **`types.ts`** — додано `userRole?: 'teacher' | 'student'` прямо в
  `LessonRoomProps` (більше не потрібне локальне розширення типу в
  `LessonRoom.tsx`).

`LessonRoomSettingsModal.tsx`, `LessonRoomChatSidebar.tsx` та `utils.ts` не
змінювались — новий функціонал їх не зачіпає.

## 4. Як це працює технічно

- **Реакції / чат / опитування / таймер / kick / mute-all** — усе йде через
  один і той самий Agora Data Stream (`sendStreamMessage` /
  `stream-message`), просто з різними `type` у JSON-повідомленні. Це
  найпростіший спосіб не піднімати окремий backend, але має нюанс: Data
  Stream **не гарантує доставку** при поганій мережі. Для чогось критичнішого
  за поточний прототип (наприклад, щоб бали з опитування точно не губились)
  варто перенести ці події на окремий надійний канал (Agora RTM, WebSocket,
  Firebase).
- **Kick / Mute all** — Agora RTC SDK не має вбудованої серверної моделі прав
  (немає способу примусово вимкнути мікрофон чи видалити іншого учасника з
  клієнта). Реалізовано кооперативно: вчительський клієнт розсилає сигнал,
  а клієнт учасника сам виконує дію (вимикає свій мікрофон / викликає
  `onLeave`). Для справжнього "неможливо оскаржити" видалення потрібен
  сервер, який відкликає токен участі через Agora REST API.
- **Список учасників з іменами** — імена інших учасників не приходять з
  Agora "з коробки" (SDK оперує лише числовими UID). Тому при підключенні
  кожен клієнт розсилає `PROFILE` (ім'я + роль), і всі інші зберігають це у
  локальному словнику `uid → {name, role}`.

## 5. Що можна покращити далі

- Перенести чат/опитування/таймер на надійніший канал (Agora RTM або власний
  WebSocket), якщо ці дані критичні.
- Персистентність: зараз усе живе тільки в пам'яті вкладки — при
  перезавантаженні сторінки історія чату, активне опитування й таймер
  губляться для цього клієнта (хоча вчитель може заново все запустити).
- Реальний "hard kick": видаляти токен на сервері через Agora REST API,
  а не покладатися лише на добровільний вихід клієнта.
